## 一、整体流程说明

微信小程序登录依赖微信官方的 **code 换凭证** 流程，整体如下：

```
小程序端                     我方后端                    微信服务器
  │                            │                            │
  │── uni.login() ────────────>│                            │
  │   获得临时 code             │                            │
  │                            │── GET jscode2session ─────>│
  │                            │   appid + secret + code    │
  │                            │<── openid + session_key ───│
  │                            │                            │
  │                            │ 查/创 SysUser（按openid）   │
  │                            │ 生成 JWT Token              │
  │<── JWT Token + 用户信息 ───│                            │
  │                            │                            │
  │ 保存 token 到 Storage       │                            │
  │ 跳转首页                   │                            │
```

**关键点：**
- `session_key` 只在服务端保存，**不下发**给前端（用于后续解密手机号等敏感数据）
- `openid` 是用户在该小程序的唯一标识，首次登录自动创建账号
- 生成的 JWT Token 格式与 H5 端完全一致，前端请求拦截器无需改动

## 二、前置准备

### 2.1 获取微信小程序 AppID 和 AppSecret

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 进入 **开发 → 开发管理 → 开发设置**
3. 记录 `AppID` 和 `AppSecret`（AppSecret 需点击"生成"并妥善保存）

### 2.2 配置服务器域名白名单

微信小程序要求后端接口域名已完成 HTTPS 配置：

1. 在公众平台进入 **开发 → 开发管理 → 服务器域名**
2. 在 `request 合法域名` 中添加你的后端域名，例如：`https://api.yourdomain.com`
3. 确保该域名已有有效 SSL 证书（Let's Encrypt 免费证书即可）

### 3.4 新增微信 API 工具类

**文件路径：** `backend/src/main/java/com/henrywang/ems/common/util/WechatApiUtil.java`

```java
package com.henrywang.ems.common.util;

import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.henrywang.ems.common.exception.BusinessException;
import com.henrywang.ems.config.WechatProperties;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.ResponseEntity;
import org.springframework.stereotype.Component;
import org.springframework.web.client.RestTemplate;
import org.springframework.web.util.UriComponentsBuilder;

@Slf4j
@Component
@RequiredArgsConstructor
public class WechatApiUtil {

    private final WechatProperties wechatProperties;
    private final ObjectMapper objectMapper;

    /**
     * 微信 jscode2session 响应结果
     */
    public record Jscode2SessionResult(String openid, String sessionKey, String unionid) {}

    /**
     * 用临时登录 code 换取 openid 和 session_key
     *
     * @param code 小程序端 uni.login() 返回的临时 code
     * @return openid、session_key、unionid
     */
    public Jscode2SessionResult jscode2session(String code) {
        String url = UriComponentsBuilder
                .fromHttpUrl(wechatProperties.getJscode2sessionUrl())
                .queryParam("appid", wechatProperties.getAppId())
                .queryParam("secret", wechatProperties.getAppSecret())
                .queryParam("js_code", code)
                .queryParam("grant_type", "authorization_code")
                .toUriString();

        log.debug("调用微信 jscode2session，code={}", code);

        try {
            RestTemplate restTemplate = new RestTemplate();
            ResponseEntity<String> response = restTemplate.getForEntity(url, String.class);
            JsonNode body = objectMapper.readTree(response.getBody());

            // 微信返回错误码
            if (body.has("errcode") && body.get("errcode").asInt() != 0) {
                int errcode = body.get("errcode").asInt();
                String errmsg = body.path("errmsg").asText("unknown error");
                log.error("微信 jscode2session 失败，errcode={}，errmsg={}", errcode, errmsg);
                throw new BusinessException("微信登录失败：" + errmsg + "（errcode=" + errcode + "）");
            }

            String openid = body.path("openid").asText();
            String sessionKey = body.path("session_key").asText();
            String unionid = body.path("unionid").asText(null); // unionid 非必有

            if (openid.isBlank()) {
                throw new BusinessException("微信登录失败：未获取到 openid");
            }

            log.info("微信登录成功，openid 前缀={}", openid.substring(0, Math.min(8, openid.length())));
            return new Jscode2SessionResult(openid, sessionKey, unionid);

        } catch (BusinessException e) {
            throw e;
        } catch (Exception e) {
            log.error("调用微信接口异常", e);
            throw new BusinessException("微信接口调用失败，请稍后重试");
        }
    }
}
```

### 3.6 在 AuthServiceImpl 中实现微信登录

**文件路径：** `backend/src/main/java/com/henrywang/ems/service/impl/AuthServiceImpl.java`

在类中**注入** `WechatApiUtil` 和 `SysUserRepository`，并实现 `wxLogin` 方法：

```java
// 在类的字段注入区域添加：
private final WechatApiUtil wechatApiUtil;

// 实现方法：
@Override
@Transactional
public LoginResponse wxLogin(WxLoginRequest request) {
    // 1. 用 code 换取 openid 和 session_key
    WechatApiUtil.Jscode2SessionResult wxResult = wechatApiUtil.jscode2session(request.getCode());
    String openid = wxResult.openid();
    String sessionKey = wxResult.sessionKey();
    String unionid = wxResult.unionid();

    // 2. 按 openid 查找用户，不存在则自动创建
    SysUser user = userRepository.findByOpenid(openid).orElseGet(() -> {
        log.info("微信新用户首次登录，自动创建账号，openid 前缀={}", openid.substring(0, 8));
        SysUser newUser = new SysUser();
        newUser.setOpenid(openid);
        newUser.setUnionid(unionid);
        newUser.setRole(UserRole.WORKER);  // 小程序端默认角色为工人
        newUser.setStatus(1);
        // 若前端传入了昵称/头像，直接使用
        if (request.getNickname() != null && !request.getNickname().isBlank()) {
            newUser.setWxNickname(request.getNickname());
            newUser.setRealName(request.getNickname());
        } else {
            newUser.setRealName("微信用户");
        }
        if (request.getAvatarUrl() != null && !request.getAvatarUrl().isBlank()) {
            newUser.setWxAvatar(request.getAvatarUrl());
            newUser.setAvatar(request.getAvatarUrl());
        }
        return userRepository.save(newUser);
    });

    // 3. 更新 session_key（每次登录刷新）和微信用户信息
    user.setSessionKey(sessionKey);
    if (unionid != null) user.setUnionid(unionid);
    if (request.getNickname() != null && !request.getNickname().isBlank()) {
        user.setWxNickname(request.getNickname());
        if (user.getRealName() == null || user.getRealName().equals("微信用户")) {
            user.setRealName(request.getNickname());
        }
    }
    if (request.getAvatarUrl() != null && !request.getAvatarUrl().isBlank()) {
        user.setWxAvatar(request.getAvatarUrl());
        if (user.getAvatar() == null) {
            user.setAvatar(request.getAvatarUrl());
        }
    }
    user.setLastLoginAt(LocalDateTime.now());
    userRepository.save(user);

    // 4. 检查账号状态
    if (user.getStatus() == null || user.getStatus() != 1) {
        throw new BusinessException("账号已被禁用，请联系管理员");
    }

    // 5. 生成 JWT Token（与 H5 端格式完全一致）
    String token = jwtUtils.generateToken(user.getId(), user.getRole().name(),
            user.getUsername() != null ? user.getUsername() : openid);

    log.info("微信用户登录成功，userId={}, role={}", user.getId(), user.getRole());

    return LoginResponse.builder()
            .token(token)
            .tokenType("Bearer")
            .expiresIn(86400L)
            .user(LoginResponse.UserInfoVO.builder()
                    .id(user.getId())
                    .username(user.getUsername())
                    .realName(user.getRealName())
                    .phone(user.getPhone())
                    .avatar(user.getAvatar() != null ? user.getAvatar() : user.getWxAvatar())
                    .role(user.getRole().name())
                    .build())
            .build();
}
```

<script setup>
// #ifdef MP-WEIXIN
import { ref } from 'vue'
import { useUserStore } from '@/store/modules/user'
import { wxLogin } from '@/api/modules/auth'

const userStore = useUserStore()
const loading = ref(false)

/**
 * 步骤1：用户点击"微信一键登录"按钮
 * open-type="getUserInfo" 会弹出授权弹窗（仅首次）
 * 授权后触发 getuserinfo 事件
 */
async function onGetUserInfo(e) {
  if (loading.value) return

  // 用户拒绝授权
  if (!e.detail || !e.detail.userInfo) {
    // 微信新版已不强制要求昵称，此处继续登录但不获取昵称
    console.log('用户未授权获取昵称，继续匿名登录')
  }

  const nickname = e.detail?.userInfo?.nickName || ''
  const avatarUrl = e.detail?.userInfo?.avatarUrl || ''

  loading.value = true

  try {
    // 步骤2：获取微信临时登录 code
    const loginResult = await new Promise((resolve, reject) => {
      uni.login({
        provider: 'weixin',
        success: resolve,
        fail: reject
      })
    })

    const code = loginResult.code
    if (!code) {
      uni.showToast({ title: '获取登录凭证失败', icon: 'error' })
      return
    }

    // 步骤3：将 code 发送到后端换取 JWT Token
    const res = await wxLogin(code, nickname, avatarUrl)

    if (res.code === 200) {
      const { token, user } = res.data

      // 步骤4：保存 token 和用户信息
      userStore.setToken(token)
      userStore.setUserInfo(user)

      uni.showToast({ title: '登录成功', icon: 'success' })

      // 步骤5：跳转首页
      setTimeout(() => {
        uni.switchTab({ url: '/pages/home/index' })
      }, 500)
    }
  } catch (error) {
    console.error('微信登录失败：', error)
    uni.showToast({ title: '登录失败，请重试', icon: 'error' })
  } finally {
    loading.value = false
  }
}

function openPrivacy() {
  // TODO: 跳转到隐私协议页面

    title: '用户隐私协议',
    content: '本系统仅收集您的微信昵称用于显示，不会向第三方泄露您的个人信息。',
    showCancel: false
  })
}
// #endif
</script>

### 6.3 常见错误处理

| 微信错误码 | 含义 | 处理方式 |
|-----------|------|---------|
| `40029` | code 无效（已使用或过期） | 重新调用 `uni.login()` |
| `45011` | 频率限制（每分钟100次） | 稍后重试 |
| `40163` | code 已被使用 | 前端避免重复调用 |
| `-1` | 微信系统繁忙 | 重试 |

## 七、安全注意事项

1. **AppSecret 绝不下发前端**：前端只持有 code，code → openid 的换取必须在后端进行
2. **session_key 安全**：仅存储在后端数据库，不写入 JWT Token，不下发前端
3. **code 单次有效**：每次登录都调用 `uni.login()` 获取新 code，不要缓存 code
4. **openid 不等于用户身份**：只用于 identify 用户，不用于敏感操作授权
5. **新用户自动注册**：默认 `WORKER` 角色，如需绑定管理员账号需另行实现绑定流程

## 八、后续扩展（可选）

### 8.1 手机号绑定（强烈推荐）

微信新版 API 支持通过 `open-type="getPhoneNumber"` 获取用户手机号，可用于：
- 将工人微信账号与管理员在系统中登记的手机号进行关联
- 实现 `workerId` 与 `SysUser` 的自动绑定

流程：
```
小程序端 open-type="getPhoneNumber"
    → 用户授权 → 获取 encryptedData + iv
    → 发送到后端 POST /api/auth/bind-phone
    → 后端用 session_key 解密得到手机号
    → 查找 worker 表中的手机号匹配
    → 设置 user.workerId
```

### 8.2 工人身份自动识别

在 `wxLogin` 成功后，可根据 `user.getPhone()` 在 `worker` 表中匹配工人，自动设置 `workerId`，使工人登录后能直接看到本人的考勤和工资条，无需手动绑定。
